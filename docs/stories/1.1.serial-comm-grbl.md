# Story 1.1: Implement and Test Serial Communication with GRBL Arduino

**Status:** Draft

**Story**
As a developer,
I want to implement and verify reliable serial communication between the controller Arduino and the GRBL Arduino,
so that GCode commands can be sent and responses received for plotter control.

**Acceptance Criteria**
- Serial1 is initialized at 115200 baud and connected to the GRBL Arduino.
- The controller can send a test message and receive a response from GRBL.
- Communication errors (timeouts, disconnects) are detected and handled gracefully.
- Code is integrated with the main state machine and does not break existing functionality.

**Tasks / Subtasks**
- [ ] Set up Serial1 for communication with GRBL.
- [ ] Write test code to send and receive messages.
- [ ] Implement error handling for communication failures.
- [ ] Document integration points and any changes to main.cpp or related files.

**Dev Notes**
- Use hardware serial (Serial1) for GRBL communication.
- Ensure that the communication does not interfere with USB Serial (Serial).
- Test with both Arduinos connected and powered.

**Testing**
- Confirm that test messages are sent and received reliably.
- Simulate disconnects and verify error handling.
- Validate that the rest of the controller functions are unaffected.
