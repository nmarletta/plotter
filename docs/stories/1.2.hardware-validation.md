# Story 1.2: Hardware Validation & Integration Smoke Test

## Status
In Progress

## Story
**As a** plotter controller developer,
**I want** a hardware validation mode that tests all peripherals and GRBL connectivity end-to-end,
**so that** I can confirm the full hardware stack (display, encoder, SD card, Serial1/GRBL, fan pin) is correctly wired and integrated before building further features on top of it.

## Story Context

**Existing System Integration:**
- Integrates with: `state_main` (adds temporary "Diagnostics" menu item), `State` enum in `globals.h`, `main.cpp` loop switch
- Technology: Arduino C++, U8g2 (OLED), RotaryButton encoder, SdFat SD, SerialManager (Serial1)
- Follows pattern: Free function `state_*()` with `bool *_active = false` activation guard, `encoder.turned()` / `encoder.pressed()` for input, `u8g2` + `centeredText()` for output
- Touch points: `globals.h` (new state enum value), `main.cpp` (new case in loop switch), `state_main.cpp` (new menu item)

## Acceptance Criteria

**Functional — Interactive OLED Diagnostic Mode (AC1–AC5):**

1. A "Diagnostics" option appears in the main menu, navigates to a new `HWTEST` state.
2. The diagnostic mode cycles through 5 sequential test screens, advancing on encoder press:
   - **Screen 1 — Display:** Shows "Display OK / Press →" (self-evident pass if visible).
   - **Screen 2 — Encoder:** Shows a counter value; rotating changes it ±1; pressing confirms and advances.
   - **Screen 3 — SD Card:** Attempts to list the root directory; shows file count or "SD FAIL".
   - **Screen 4 — GRBL Serial:** Sends a `\n` wake to Serial1; shows first non-empty response line within 2 seconds, or "NO GRBL" on timeout. `serialMgr` is used for this.
   - **Screen 5 — Fan Pin:** Shows "FAN OFF"; encoder press toggles `PIN_FAN` HIGH/LOW and updates label; long-press (>1s) exits back to `MAIN`.
3. Each screen displays a step indicator (e.g., "1/5", "2/5"…) in the top-right corner.
4. No screen blocks indefinitely — SD and GRBL screens timeout gracefully (≤2 seconds) and show failure message rather than hanging.
5. Pressing back/long-press from any screen returns to `MAIN` state cleanly, resetting `hwtest_active`.

**Functional — Automated pio Tests (AC6–AC7):**

6. Existing `test/test_serial_manager/test_serial.cpp` parser and timeout tests all pass on hardware via `pio test`.
7. A new `test/test_serial_manager/test_serial_grbl.cpp` file contains GRBL-connected integration tests (skipped gracefully if no GRBL device detected within 500ms):
   - Send `\n` to wake GRBL; verify a non-empty response is received.
   - Send `$$` command; verify response contains at least one `$` settings line.
   - Verify `serialMgr.isReady()` returns `true` after `begin()`.

**Integration (AC8):**

8. After this story, the existing file browser, display rendering, and encoder navigation remain fully functional (no regression).

## Dev Notes

### Architecture Context
This story is a **validation gate** — it must pass before Story 1.3 (ProgressTracker) and Story 1.4 (GCodeStreamer validation) begin. Its purpose is to surface wiring problems (wrong pins, missing init, SD not mounting, GRBL not responding) as early and cheaply as possible.

### Key Implementation Patterns
- Follow `state_main.cpp` exactly for activation guard pattern (`bool hwtest_active`)
- Use `millis()` for all timeouts — never `delay()` inside a state function
- For the GRBL screen, use `Serial1` directly (not `serialMgr.sendLine()`) to avoid the 1s `waitForOk` blocking the display; just peek for any byte within 2s
- `encoder.pressed()` returns true on release; for long-press detection, track `pressStart = millis()` on first detection of button down, then check delta on release
- `PIN_FAN` is defined in `main.cpp` — either `extern const int PIN_FAN` or redefine it as `2` in the test state (check `main.cpp`)

### Screen Layout Reference (128×64 OLED)
```
+------------------------+---+
| <title>          <N/5> |   |
|                        |   |
| <status line 1>        |   |
| <status line 2>        |   |
| <hint: Press ->  >     |   |
+------------------------+---+
```
Font: `u8g2_font_6x13_tf` (same as `displayList`); step indicator at x=104, y=11.

### GRBL Wake Sequence
GRBL 1.1f requires a soft-reset (`0x18`) or newline to begin responding after power-on. Send `Serial1.println("")` (empty line) then poll `Serial1.available()` in a millis() loop. First line may be `Grbl 1.1f ['$' for help]` — any non-empty line is a pass.

### Fan Pin
`PIN_FAN` is `#define PIN_FAN 2` in `main.cpp`. Since it's a `#define`, it's globally visible. Use `digitalWrite(PIN_FAN, HIGH/LOW)` directly.

### Risk
- **SD card not mounting in HWTEST state**: SD is initialised in `setup()` in `main.cpp` — `fileHandler.loadFileNames()` should just work; if not, the screen will show "SD FAIL" which is the intended failure path.
- **GRBL not connected**: Handled gracefully with "NO GRBL" display and 2s timeout. This is expected during initial dev without GRBL hardware.
