# Story 1.2: Design and Implement GCode Sending Module

**Status:** Draft

**Story**
As a developer,
I want to design and implement a robust GCode sending module/class,
so that the controller can reliably stream GCode commands from SD card to the GRBL Arduino and handle responses and errors.

**Acceptance Criteria**
- A dedicated GCode sender class/module exists and is integrated with the project.
- The module can read GCode lines from SD card and send them to GRBL via Serial1.
- The module handles GRBL responses, including OK, error, and busy states.
- Errors and communication issues are logged and handled gracefully.
- The module can be triggered from the PLOT state.

**Tasks / Subtasks**
- [ ] Design the GCode sender class/module interface.
- [ ] Implement reading GCode lines from SD card.
- [ ] Implement sending GCode lines to GRBL and handling responses.
- [ ] Integrate error handling and logging.
- [ ] Connect the module to the PLOT state in the state machine.
- [ ] Document the module and its integration points.

**Dev Notes**
- Reuse or refactor any existing GCodeStreamer code if present.
- Ensure non-blocking operation where possible to keep UI responsive.
- Consider edge cases: file end, GRBL busy, communication loss.

**Testing**
- Test with real GCode files and GRBL hardware.
- Simulate errors and verify error handling.
- Confirm that the module works as expected when triggered from the UI.
